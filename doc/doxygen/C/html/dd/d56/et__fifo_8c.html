Detailed Description

These routines use an ET system as a FIFO. The idea is that an ET system divides its events (or buffers) into groups (or arrays) of fixed size. Each array is then a single FIFO entry. Thus the FIFO's entries are each an array of related buffers.

When requesting an entry, this interface does so in arrays of this fixed size. Likewise, when putting them back it does so only in the same arrays. In order to use this interface, the ET system should be run using the et_start_fifo program and not et_start. This sets up the ET to be comprised of properly grouped events. It also creates one station, called "Users" to which the consumers of FIFO entries will attach.

This interface is intended to be used by connecting to a local ET system using shared memory - providing a fast, wide FIFO available to multiple local processes. Although it could be used by remote users of the ET system, it's advantages would be lost.

These routines are threadsafe with the exception of et_fifo_getBuf, but that should never be called on the same arg with different threads. These routines are designed to work as follows:

  • et_open to open ET system and get id
  • et_fifo_openProducer or et_fifo_openConsumer to get fifo id as either a producer or consumer of entries
    • Makes sure ET is setup properly
    • Attaches to GrandCentral station if producing entries
    • Attaches to Users station if consuming entries
  • et_fifo_newEntry to get pointer to et_fifo_entry if producing fifo entry
  • et_fifo_newEntryTO to get pointer to et_fifo_entry if producing fifo entry
  • et_fifo_getEntry to get pointer to et_fifo_entry if consuming fifo entry
  • et_fifo_getEntryTO to get pointer to et_fifo_entry if consuming fifo entry
  • et_fifo_putEntry when finished with fifo entry
  • et_fifo_close when finished with fifo interface
  • et_close when finished with ET system
  • For operations on a fifo entry:
    • et_fifo_getBufs to get access to all the events of a single fifo entry
    • et_fifo_getBuf to get access to a single event of a single fifo entry
  • For operations on fifo in general:
    • et_fifo_getEntryCapacity to get the number of events in array. Need only be done once.
    • et_fifo_getBufSize to get available size of each event. Need only be done once.
    • et_fifo_getIdCount to get the number of buffers assigned an id in each entry.
    • et_fifo_getBufIds to get the array of ids assigned to buffers in each entry.
    • et_fifo_getEntryCount to get the total number of entries existing in the fifo.
    • et_fifo_getFillLevel to get the number of fifo entries containing unconsumed data, which is the fill level of the fifo.
  • For operations on a specific event/buffer in an entry:
    • et_fifo_getId to get an id for a particular event
    • et_fifo_setId to set an id for a particular event
    • et_fifo_setHasData to set whether this event has data in it or not.
    • et_fifo_hasData to get whether this ET event has data in it or not.
    • et_event_getlength to get the size of valid data for a particular event

The first integer of an event's control array is reserved to hold an id to distinguish it from the other events. It's initially set to -1. Note that once you call et_fifo_getBufs and get pointers to the events, you have full access to the control array of each event by calling et_event_getcontrol and et_event_setcontrol and are not restricted to using et_fifo_getId and et_fifo_setId which only access the first integer of the control array. The control array is of size ET_STATION_SELECT_INTS.
